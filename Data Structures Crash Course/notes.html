Data Structures Crash Course Notes

Memory

  • All memory in a system is limited, where you cannot add more memory if you run out. Use and free memory wisely!
  • Every variable takes up a memory slot
  • Memory is represented by bits
    • Every 8 bits is 1 byte
    • An integer that is 32-bit takes up 1 slot, while a 64-bit integer takes up 2 slots
  • Endianness refers to the order of bytes, big or small endian does not matter as long as it is consistent
  • Strings are stored in memory as ASCII codes, to keep each letter/symbol binary
    • Binary numbers are just the 1s and 0s that the computer can read, it starts with the base of 2 to the power of 1.
  • Pointers just point to a memory address to access the data without having to "re-store" it in the memory

Logarithm

  • Although log(n) time may be daunting at first, it can be compared to the opposite of a power of 2 that tips in your favor
  • Log(n) time happens when an algorithm cuts the input in half every time it runs
  • It can also be rewritten as log(4) = 2 or 22 = 4
